Inferring VHDL FIR filters in Vivado

Posted on Sun 14 August 2016 in VHDL, Vivado

Xilinx provides an FIR Compiler IP core which does a fantastic job of creating FIR filers. You should definitely use it, particularly for multi-rate filters where it can efficiently pack DSP usage by timeslicing their usage. However, for creating many filters with different coefficients, if you wants portable code or if you just want to see how it all works then you have to check that the synthesis engine is making the best use of the hardware -- in this case the DSP48E1 slices -- hopefully from generic VHDL and not having to manually instantiate the DSP slice.

The DSP48E1 slices have heaps of functionality (user guide).
